# Spatial Server API

> **Last Updated:** 2026-06-26

## Purpose

Define the complete API contract between the Business Backend and Spatial Server.

## Transport

- Protocol: gRPC over HTTP/2
- Service: `SpatialServerAPI` (defined in proto/spatial_server_api.proto)
- Addressing: Room Service gRPC endpoint (port 9000, private network)
- Auth: mTLS or shared API secret (configurable per deployment)
- Timeout: 10s default, configurable per RPC

## Authentication Model

- Spatial Server never issues tokens
- Spatial Server never manages users
- Business Backend issues JWT runtime tokens
- Business Backend provides public key for token validation
- Token contents: player_id, runtime_id, exp
- Token TTL: 15 minutes (access), 24 hours (refresh — managed by Business Backend)

## Runtime Lifecycle API

The following methods are exposed by Room Service (gRPC) for Business Backend consumption.

| Method | Description |
|--------|-------------|
| `CreateRuntime` | Allocate zones and Game Servers for a new runtime |
| `DestroyRuntime` | Release all resources for a runtime |
| `GetRuntimeInfo` | Query runtime status, player count, zone count |
| `GetRuntimeMetrics` | Query runtime performance metrics |
| `ListRuntimes` | List all active runtimes |

### CreateRuntime

Allocates zones and assigns Game Servers for a new runtime. Called when a player creates a room/showroom.

**Request:**
| Field | Type | Description |
|-------|------|-------------|
| runtime_id | string | UUIDv7, generated by the caller |
| zone_count | uint32 | Number of zones to allocate |

**Response:**
| Field | Type | Description |
|-------|------|-------------|
| runtime_id | string | Echo of the request ID |
| gateway_addr | string | Host:port of the assigned Gateway |
| zones | repeated ZoneInfo | Zone descriptors (id, server_addr) |

**Error codes:** ALREADY_EXISTS (duplicate runtime_id), NO_SERVER_AVAILABLE, RATE_LIMITED

### DestroyRuntime

Releases all resources (zones, Game Servers) for a runtime. Called when a room/showroom ends.

**Request:**
| Field | Type | Description |
|-------|------|-------------|
| runtime_id | string | UUIDv7 of the runtime to destroy |

**Response:** Empty.

**Error codes:** NOT_FOUND, RUNTIME_ALREADY_DESTROYED, TRANSFER_IN_PROGRESS

### GetRuntimeInfo

Returns current status and metadata for a runtime.

**Request:**
| Field | Type | Description |
|-------|------|-------------|
| runtime_id | string | UUIDv7 of the runtime |

**Response:**
| Field | Type | Description |
|-------|------|-------------|
| runtime_id | string | Echo of the request ID |
| state | RuntimeState | Current lifecycle state |
| player_count | uint32 | Connected players across all zones |
| zone_count | uint32 | Active zones |
| created_at | Timestamp | When the runtime was created |

**Error codes:** NOT_FOUND

### GetRuntimeMetrics

Returns performance metrics for a runtime.

**Request:**
| Field | Type | Description |
|-------|------|-------------|
| runtime_id | string | UUIDv7 of the runtime |

**Response:**
| Field | Type | Description |
|-------|------|-------------|
| runtime_id | string | Echo of the request ID |
| cpu_usage | double | Aggregate CPU usage (0.0–1.0) |
| memory_bytes | uint64 | Aggregate memory usage |
| network_rx_bytes | uint64 | Total bytes received |
| network_tx_bytes | uint64 | Total bytes sent |
| tick_rate_hz | double | Current simulation tick rate |

**Error codes:** NOT_FOUND

### ListRuntimes

Lists all active runtimes (creating and active states).

**Request:** Empty.

**Response:**
| Field | Type | Description |
|-------|------|-------------|
| runtimes | repeated RuntimeInfo | List of active runtime descriptors |

**Error codes:** None.

## Runtime States

| State | Description |
|-------|-------------|
| `creating` | Zones being allocated, Game Servers being assigned |
| `active` | Runtime is running, players can connect |
| `draining` | Runtime is being destroyed, players being disconnected |
| `destroyed` | All resources released |

## Runtime Flow

1. **Business Backend** creates a room/showroom in its own database.
2. **Business Backend** calls Spatial Server: `CreateRuntime(runtime_id, zone_count)`.
3. **Room Service** allocates zones, assigns Game Servers, stores runtime metadata in PostgreSQL.
4. **Business Backend** receives `gateway_addr` and returns it to the client.
5. **Client** connects to Gateway with runtime token (JWT containing: player_id, runtime_id, exp).
6. **Gateway** validates token signature → looks up runtime zone → proxies to Game Server.
7. **Client** participates in realtime simulation.
8. When room ends, **Business Backend** calls `DestroyRuntime(runtime_id)`.
9. **Room Service** marks runtime as destroyed, releases zones, disconnects players.

## Ownership

- Runtime is owned by the Business Backend that created it.
- Spatial Server does not enforce multi-tenant isolation (trusted Business Backend model).

## Error Model

### gRPC Error Codes

| gRPC Code | Map to | Description |
|-----------|--------|-------------|
| NOT_FOUND | ZONE_NOT_FOUND, SERVER_NOT_FOUND, RUNTIME_NOT_FOUND | Resource not found |
| ALREADY_EXISTS | RUNTIME_ALREADY_EXISTS | Resource already exists |
| UNAVAILABLE | NO_SERVER_AVAILABLE | Service temporarily unavailable |
| FAILED_PRECONDITION | ZONE_NOT_OWNED, TRANSFER_IN_PROGRESS, SERVER_DRAINING | Operation not allowed in current state |
| RESOURCE_EXHAUSTED | RATE_LIMITED | Rate limit exceeded |
| INVALID_ARGUMENT | | Invalid request parameters |

### Error Response Format

```json
{
    "code": "RUNTIME_NOT_FOUND",
    "message": "Runtime runtime-abc123 not found",
    "details": {}
}
```

## Versioning Strategy

- SpatialServerAPI is versioned via gRPC package (spatial.v1, spatial.v2)
- Breaking changes increment the major version
- Backward-compatible additions are allowed within a major version
- Deprecated RPCs remain available for one major version cycle

## Consumer Notes

- Business Backend must generate UUIDv7 for runtime_id
- Zone count should be estimated based on expected concurrent players (100 players per zone)
- Gateway address returned may be a load balancer address (multiple Gateway instances behind it)
- Runtime cleanup: always call DestroyRuntime to avoid resource leaks
